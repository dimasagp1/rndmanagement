The root is a standard Laravel application whose `bootstrap/app.php` registers the `role`, `permission`, and `role_or_permission` middleware aliases from Spatie\Permission, and `routes/web.php` groups every feature under `auth`+`verified`. Each child feature owns its own controllers/models/migrations/views but all routes are declared centrally in this file using resource routes plus explicit action routes (`submit`, `approve`, `reject`, `print`) gated by `can:` or `role:` middleware. Cross-feature coordination happens through shared domain primitives: the `User` model with roles/permissions, the global `Setting` helper (`app/Helpers/setting.php`), and the Approval Center controller which dispatches approvals across Formula/TrialRM/TrialPM models. Blade layouts (`AppLayout`/`GuestLayout`) and reusable components in `resources/views/components` provide the common UI layer consumed by every feature's views.
