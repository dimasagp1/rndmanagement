This leaf module is the application's runtime shell — it wires together the framework without containing business logic.

- `public/index.php` is the HTTP front controller: defines `LARAVEL_START`, loads Composer's autoloader, requires `bootstrap/app.php`, and delegates to `$app->handleRequest(Request::capture())`.
- `bootstrap/app.php` constructs the `Application` via `Application::configure(...)->withRouting()->withMiddleware()->withExceptions()->create()`. It registers Spatie Permission middleware aliases (`role`, `permission`, `role_or_permission`) and forces JSON responses on `api/*` requests. `bootstrap/providers.php` lists only `App\Providers\AppServiceProvider`, deferring all third-party provider registration to that class.
- `routes/` splits routing by concern: `web.php` groups authenticated routes under `['auth','verified']` and applies Spatie's `can:` / `role:` middleware per resource; `auth.php` handles guest/authenticated auth flows (register, login, password reset, email verification); `console.php` is the Artisan command route file.
- `config/*.php` are standard Laravel environment-driven configs (`env()` with defaults) covering app identity, database (sqlite/mysql/mariadb/pgsql/sqlsrv + Redis), cache, queue, session, filesystems, logging, mail, activitylog, and permission packages.
- `storage/` holds writable runtime artifacts: compiled Blade views in `framework/views/`, cached data/sessions/testing fixtures under `framework/{cache,sessions,testing}`, user uploads under `app/public/signatures`, and `logs/laravel.log`; every writable subdirectory ships a `.gitignore` so nothing is committed.
- `bootstrap/cache/packages.php` and `services.php` are generated by `artisan package:discover` / `artisan config:cache` and are not hand-edited.
