Three-file MVC leaf: `SupplierController` extends the application base `Controller` and exposes standard RESTful actions (`index`, `create`, `store`, `edit`, `update`, `destroy`) backed by route names under the `suppliers.*` namespace; views are rendered via Blade templates in the `suppliers.*` view namespace. The `Supplier` Eloquent model declares `$fillable` fields matching the `suppliers` migration schema and integrates Spatie's `LogsActivity` trait to audit only dirty changes on a subset of columns, exposing a `hasMany(FormulaMaterial::class)` relationship that ties suppliers into the formula/materials domain. The migration defines the `suppliers` table with an auto-incrementing `id`, indexed `name` (enforced unique at the controller validation layer), optional contact/phone/email/address text fields, and Laravel timestamps.
