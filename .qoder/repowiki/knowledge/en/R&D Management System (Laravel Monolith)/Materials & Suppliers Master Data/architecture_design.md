Two parallel, symmetric CRUD modules share no code — each has its own Controller/Model/Migration/Factory pair (`MaterialController`+`Material`, `SupplierController`+`Supplier`) and is wired independently via standard Laravel resource routes. The only cross-cutting artifact is `MaterialSupplierSeeder`, which seeds both tables together so the two master datasets are always available as a unit for downstream formula/purchasing features. Views follow the convention `materials.*` / `suppliers.*` and flash messages use Indonesian labels, keeping the UI language consistent across both resources.
