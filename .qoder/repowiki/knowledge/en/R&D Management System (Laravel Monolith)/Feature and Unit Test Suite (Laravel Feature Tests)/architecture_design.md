Organised by the standard Laravel tests layout: TestCase.php extends Illuminate\Foundation\Testing\TestCase as the shared base; Unit/ holds pure-PHP unit tests extending PHPUnit\Framework\TestCase; Feature/ holds HTTP-level integration tests extending Tests\TestCase, with an Auth/ sub-namespace grouping authentication scenarios. Each feature test class that touches the database uses the RefreshDatabase trait to wrap runs in transactions, and sets up domain fixtures (roles via RolePermissionSeeder, users via User::factory()) inside setUp() or individual methods. Role checks are exercised through Spatie's assignRole / hasRole helpers on App\Models\User. There is no custom application bootstrap — the suite relies on the default Laravel phpunit.xml configuration.
